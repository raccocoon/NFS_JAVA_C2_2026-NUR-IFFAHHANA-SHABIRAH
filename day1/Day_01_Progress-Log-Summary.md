
# Day 01 Progress Log Summary

## Overview
Today’s session starts with foundational web development concepts including HTTP, JSON, APIs, and how browsers communicate with servers then the basics of full-stack development using Java and Git. The focus was on understanding object-oriented programming (OOP) concepts and how classes interact in a system.


## Topics Covered

### 1. Web Fundamentals
- Client-server architecture
- How a browser loads a webpage (URL → request → response → render)
- Role of frontend and backend systems
- Full-stack architecture overview

---

### 2. HTTP & APIs
- HTTP methods:
  - GET (retrieve data)
  - POST (create data)
  - PUT (update data)
  - DELETE (remove data)
- Request structure (method, endpoint, headers, body)
- Response structure (status code, headers, body)
- Common status codes (200, 201, 400, 401, 404, 500)

---

### 3. JSON
- JSON structure (objects, arrays, key-value pairs)
- Used as standard API response format
- Lightweight and readable compared to XML
- Example API response from real endpoints

### 4. Git Version Control (Theory)
- Purpose of Git in tracking changes
- Working directory → staging → commit → remote repository
- Basic workflow:
  - git add
  - git commit
  - git push
- Importance of version history and collaboration

### 5. Git & Project Setup
